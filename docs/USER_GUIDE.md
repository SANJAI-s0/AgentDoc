# User Guide

This guide focuses on smooth day-to-day usage for customer and reviewer personas.

## 1. Customer Flow
1. Sign in with a customer account.
2. Open `Documents` and upload a file.
3. Provide optional hints (document type, notes) to improve agent confidence.
4. Track status in `Dashboard` and `Documents`.
5. If sent to review, wait for reviewer action and final status update.

## 2. Reviewer Flow
1. Sign in with a reviewer/admin account.
2. Open `Reviews` to see pending queues.
3. Compare source document and extracted JSON.
4. Correct fields when needed and submit `approve` or `reject`.
5. Verify audit trail from document history.

## 3. Search and Traceability
1. Use `Search` with natural language terms (`invoice total`, `shipping tracking`, `kyc id`).
2. Filter by status for targeted operational follow-up.
3. Use audit endpoints to trace agent decisions end-to-end.

## 4. Best Practices for Better Accuracy
- Upload high-resolution scans where possible.
- Add a document type hint when known.
- For handwritten forms, keep text regions clear and avoid shadows.
- Use consistent file naming for easier operational search.

## 5. Accessibility and Usability Notes
- Keep key actions (upload, review decision) visible above fold.
- Use short reviewer comments with clear reason codes.
- Prefer explicit decision language: `approved`, `needs clarification`, `rejected`.
