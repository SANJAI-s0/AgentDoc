# AgentDoc Deployment Guide - Render Free Tier

This guide covers deploying AgentDoc on Render's free tier with MongoDB Atlas.

## Architecture Overview

- **Landing Page**: Static site (HTML/CSS/JS)
- **Backend API**: Python/Django with Gunicorn
- **Frontend App**: React with Vite
- **Database**: MongoDB Atlas (Free M0 cluster)
- **Storage**: Local filesystem (no MinIO needed)

## Prerequisites

1. GitHub account with this repository
2. Render account (free): https://render.com
3. MongoDB Atlas account (free): https://www.mongodb.com/cloud/atlas
4. Google Gemini API key: https://makersuite.google.com/app/apikey

## Step 1: Setup MongoDB Atlas

1. Create a free M0 cluster at https://cloud.mongodb.com
2. Create a database user with password
3. Whitelist all IPs (0.0.0.0/0) for Render access
4. Get your connection string:
   ```
   mongodb+srv://<username>:<password>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority
   ```

## Step 2: Deploy on Render

### Option A: Using Blueprint (Recommended)

1. Fork/clone this repository to your GitHub
2. Go to Render Dashboard: https://dashboard.render.com
3. Click "New" → "Blueprint"
4. Connect your GitHub repository
5. Render will detect `render.yaml` and create 3 services:
   - `agentdoc-landing` (Static Site)
   - `agentdoc-backend` (Web Service)
   - `agentdoc-frontend` (Web Service)

### Option B: Manual Setup

#### Landing Page (Static Site)

1. New → Static Site
2. Connect repository
3. Settings:
   - **Name**: agentdoc-landing
   - **Build Command**: (leave empty)
   - **Publish Directory**: `landing-page`
4. Deploy

#### Backend API (Web Service)

1. New → Web Service
2. Connect repository
3. Settings:
   - **Name**: agentdoc-backend
   - **Environment**: Python 3
   - **Build Command**:
     ```bash
     cd backend && pip install -r requirements.txt && python manage.py collectstatic --no-input && python manage.py migrate
     ```
   - **Start Command**:
     ```bash
     cd backend && python manage.py seed_demo --skip-if-present && gunicorn config.wsgi:application --bind 0.0.0.0:$PORT
     ```
   - **Plan**: Free

4. Environment Variables:
   ```
   DJANGO_SECRET_KEY=<auto-generated>
   DJANGO_DEBUG=0
   DJANGO_ALLOWED_HOSTS=<your-backend-url>.onrender.com
   DJANGO_CORS_ALLOWED_ORIGINS=https://<your-frontend-url>.onrender.com
   DJANGO_CSRF_TRUSTED_ORIGINS=https://<your-frontend-url>.onrender.com
   GEMINI_API_KEY=<your-gemini-key>
   MONGODB_URI=<your-mongodb-atlas-uri>
   MONGODB_DB_NAME=agentdoc
   USE_LOCAL_STORAGE=1
   DJANGO_SESSION_COOKIE_SECURE=1
   DJANGO_CSRF_COOKIE_SECURE=1
   ```

#### Frontend App (Web Service)

1. New → Web Service
2. Connect repository
3. Settings:
   - **Name**: agentdoc-frontend
   - **Environment**: Node
   - **Build Command**:
     ```bash
     cd frontend && npm install && npm run build
     ```
   - **Start Command**:
     ```bash
     cd frontend && npm run preview -- --host 0.0.0.0 --port $PORT
     ```
   - **Plan**: Free

4. Environment Variables:
   ```
   VITE_API_BASE=https://<your-backend-url>.onrender.com/api
   NODE_VERSION=20.x
   ```

## Step 3: Update Landing Page

After deployment, update the demo link in `landing-page/js/config.js`:

```javascript
const CONFIG = {
    demoUrl: 'https://<your-frontend-url>.onrender.com',
    // ...
};
```

Commit and push to trigger redeployment.
