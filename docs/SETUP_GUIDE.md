# AgentDoc Setup Guide

Complete setup guide for local development and Render deployment.

## Project Structure

```
AgentDoc/
├── landing-page/          # Static landing page (HTML/CSS/JS + GSAP)
├── backend/              # Django API + 5-agent system
├── frontend/             # React application
├── render.yaml           # Render deployment blueprint
├── QUICKSTART.md         # Quick deployment guide
└── SETUP_GUIDE.md        # This file
```

## 5-Agent System

The system uses 5 specialized agents:

1. **Classification Agent** - Identifies document type with confidence scoring
2. **Extraction Agent** - Extracts structured data using AI-powered OCR
3. **Validation Agent** - Validates against business rules and semantic checks
4. **Routing Agent** - Routes to auto-approve, review, or exception handling
5. **Audit Agent** - Creates immutable audit trail for compliance

## Local Development Setup

### Prerequisites

- Python 3.11+
- Node.js 20+
- MongoDB (local or Atlas)
- Google Gemini API key

### Step 1: Clone Repository

```bash
cd AgentDoc
```

### Step 2: Backend Setup

```bash
cd backend

# Create virtual environment
python -m venv venv

# Activate (Windows)
venv\Scripts\activate

# Activate (Mac/Linux)
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Create .env file in AgentDoc root
```

Create `.env` file:
```env
# Required
GEMINI_API_KEY=your_gemini_api_key_here
MONGODB_URI=mongodb://localhost:27017

# Optional (defaults shown)
MONGODB_DB_NAME=agentdoc
DJANGO_DEBUG=1
DJANGO_SECRET_KEY=dev-secret-key-change-in-production
DJANGO_ALLOWED_HOSTS=localhost,127.0.0.1
DJANGO_CORS_ALLOWED_ORIGINS=http://localhost:5173,http://localhost:8080
DJANGO_CSRF_TRUSTED_ORIGINS=http://localhost:5173,http://localhost:8080
USE_LOCAL_STORAGE=1
```

```bash
# Run migrations
python manage.py migrate

# Seed demo data
python manage.py seed_demo

# Start backend
python manage.py runserver
```

Backend will run at: http://localhost:8000

### Step 3: Frontend Setup

```bash
cd ../frontend

# Install dependencies
npm install

# Create .env file
echo "VITE_API_BASE=http://localhost:8000/api" > .env

# Start frontend
npm run dev
```

Frontend will run at: http://localhost:5173

### Step 4: Landing Page

```bash
cd ../landing-page

# Serve with Python
python -m http.server 8080

# Or with Node
npx serve -p 8080
```

Landing page will run at: http://localhost:8080

## Render Deployment

### Prerequisites

1. GitHub account with this repository
2. Render account (free): https://render.com
3. MongoDB Atlas account (free): https://cloud.mongodb.com
4. Google Gemini API key: https://makersuite.google.com/app/apikey

### Step 1: Setup MongoDB Atlas

1. Create free M0 cluster at https://cloud.mongodb.com
2. Database Access → Add Database User
3. Network Access → Add IP Address → Allow Access from Anywhere (0.0.0.0/0)
4. Get connection string: `mongodb+srv://username:password@cluster.mongodb.net/`

### Step 2: Deploy on Render

1. Go to Render Dashboard: https://dashboard.render.com
2. Click "New" → "Blueprint"
3. Connect your GitHub repository
4. Render will detect `render.yaml` and create 3 services:
   - `agentdoc-landing` (Static Site)
   - `agentdoc-backend` (Web Service)
   - `agentdoc-frontend` (Web Service)

### Step 3: Configure Environment Variables

#### Backend Service

Go to backend service → Environment:

```
GEMINI_API_KEY=<your-gemini-api-key>
MONGODB_URI=<your-mongodb-atlas-connection-string>
MONGODB_DB_NAME=agentdoc
DJANGO_SECRET_KEY=<auto-generated-by-render>
DJANGO_DEBUG=0
DJANGO_ALLOWED_HOSTS=<your-backend-url>.onrender.com
DJANGO_CORS_ALLOWED_ORIGINS=https://<your-frontend-url>.onrender.com,https://<your-landing-url>.onrender.com
DJANGO_CSRF_TRUSTED_ORIGINS=https://<your-frontend-url>.onrender.com,https://<your-landing-url>.onrender.com
USE_LOCAL_STORAGE=1
MEDIA_ROOT=/opt/render/project/src/AgentDoc/backend/media
DJANGO_SESSION_COOKIE_SECURE=1
DJANGO_CSRF_COOKIE_SECURE=1
DJANGO_USE_X_FORWARDED_HOST=1
DJANGO_TRUST_X_FORWARDED_PROTO=1
```

#### Frontend Service

Go to frontend service → Environment:

```
VITE_API_BASE=https://<your-backend-url>.onrender.com/api
NODE_VERSION=20.x
```

#### Landing Page Service

No environment variables needed (static site).

### Step 4: Update Landing Page Demo Link

After deployment, update `landing-page/js/config.js`:

```javascript
const CONFIG = {
    demoUrl: 'https://<your-frontend-url>.onrender.com',
    // ...
};
```

Commit and push to trigger redeployment.

### Step 5: Test Deployment

1. Visit landing page: `https://<landing-url>.onrender.com`
2. Click "Launch Demo"
3. Login with demo credentials:
   - Customer: `customer_demo / DemoPass123!`
   - Reviewer: `reviewer_demo / DemoPass123!`

## Troubleshooting

### Backend won't start

- Check MongoDB connection string
- Verify Gemini API key is valid
- Check environment variables
- Review logs: `python manage.py runserver --verbosity 3`

### Frontend can't connect to backend

- Verify CORS settings in backend
- Check VITE_API_BASE is correct
- Ensure backend is running
- Check browser console for errors

### Landing page demo link not working

- Update `landing-page/js/config.js` with correct frontend URL
- Clear browser cache
- Check browser console for errors

### Build timeout on Render

- Remove unused dependencies
- Use `--no-cache-dir` flag for pip
- Check build logs for errors
- Consider upgrading to paid tier

### Database connection fails

- Whitelist 0.0.0.0/0 in MongoDB Atlas
- Check connection string format
- Verify database user credentials
- Test connection locally first

## Free Tier Limitations

- **Spin Down**: Services sleep after 15 minutes of inactivity
- **Cold Start**: First request after sleep takes 30-60 seconds
- **Storage**: Local storage is ephemeral (resets on redeploy)
- **Build Time**: 15 minutes max per build

### Workarounds

1. **Keep Alive**: Use UptimeRobot to ping services every 14 minutes
2. **Persistent Storage**: Use MongoDB GridFS for file storage
3. **Build Optimization**: Remove unused dependencies

## API Endpoints

### Authentication
- `POST /api/auth/login/` - Login
- `POST /api/auth/logout/` - Logout
- `GET /api/auth/me/` - Get current user

### Documents
- `POST /api/documents/upload-url/` - Get presigned upload URL
- `POST /api/documents/upload/` - Upload document
- `GET /api/documents/` - List documents
- `GET /api/documents/{id}/` - Get document details

### Reviews
- `GET /api/reviews/` - List reviews
- `POST /api/reviews/{id}/submit/` - Submit review decision

### Search
- `GET /api/search/semantic/?q=query` - Semantic search
