# Integrated Deployment Guide

## Overview

AgentDoc now uses a single integrated backend that serves both the landing page and the application frontend. This simplifies deployment and reduces costs by eliminating the need for separate frontend services.

## Architecture

```
┌─────────────────────────────────────────┐
│         Single Django Service           │
├─────────────────────────────────────────┤
│  Landing Page (/)                       │
│  Application (/app)                     │
│  API Endpoints (/api)                   │
│  Admin Panel (/admin)                   │
│  Static Files (/static)                 │
│  Media Files (/media)                   │
└─────────────────────────────────────────┘
```

## Benefits

1. **Simplified Deployment**: Single service to deploy and manage
2. **Cost Effective**: Uses only one Render free tier instance
3. **No CORS Issues**: Frontend and backend on same domain
4. **Better Security**: Reduced attack surface
5. **Easier Maintenance**: Single codebase to update

## Local Development

### Prerequisites

- Python 3.11+
- MongoDB (local or Atlas)
- Google Gemini API key

### Setup Steps

1. **Clone and Navigate**
   ```bash
   cd AgentDoc/backend
   ```

2. **Create Virtual Environment**
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```

3. **Install Dependencies**
   ```bash
   pip install -r requirements.txt
   ```

4. **Configure Environment**
   ```bash
   cp ../.env.example ../.env
   # Edit .env with your settings
   ```

5. **Run Migrations**
   ```bash
   python manage.py migrate
   ```

6. **Collect Static Files**
   ```bash
   python manage.py collectstatic --noinput
   ```

7. **Seed Demo Data (Optional)**
   ```bash
   python manage.py seed_demo
   ```

8. **Start Development Server**
   ```bash
   python manage.py runserver
   ```

9. **Access Application**
   - Landing Page: http://localhost:8000/
   - Application: http://localhost:8000/app/
   - API: http://localhost:8000/api/
   - Admin: http://localhost:8000/admin/

## Production Deployment (Render)

### Step 1: Prepare MongoDB

1. Create MongoDB Atlas account (free tier)
2. Create a cluster
3. Create database user
4. Whitelist Render IP addresses (or allow all: 0.0.0.0/0)
5. Get connection string

### Step 2: Prepare Repository

1. Push code to GitHub
2. Ensure `.env` is in `.gitignore`
3. Verify `render.yaml` is present

### Step 3: Deploy to Render

#### Option A: Using Blueprint (Recommended)

1. Go to [Render Dashboard](https://dashboard.render.com/)
2. Click "New" → "Blueprint"
3. Connect your GitHub repository
4. Select the repository
5. Render will detect `render.yaml`
6. Configure environment variables:
   - `DJANGO_ALLOWED_HOSTS`: your-app.onrender.com
   - `DJANGO_CORS_ALLOWED_ORIGINS`: https://your-app.onrender.com
   - `DJANGO_CSRF_TRUSTED_ORIGINS`: https://your-app.onrender.com
   - `GEMINI_API_KEY`: your-gemini-api-key
   - `MONGODB_URI`: your-mongodb-connection-string
   - `BACKEND_URL`: https://your-app.onrender.com
   - `REVIEW_PORTAL_URL`: https://your-app.onrender.com/app/reviews
7. Click "Apply"

#### Option B: Manual Setup

1. Go to Render Dashboard
2. Click "New" → "Web Service"
3. Connect GitHub repository
4. Configure:
   - **Name**: agentdoc
   - **Environment**: Python
   - **Region**: Oregon (or nearest)
   - **Branch**: main
   - **Root Directory**: backend
   - **Build Command**:
     ```bash
     pip install --upgrade pip
     pip install -r requirements.txt
     python manage.py collectstatic --no-input
     python manage.py migrate
     ```
   - **Start Command**:
     ```bash
     python manage.py seed_demo --skip-if-present
     gunicorn config.wsgi:application --bind 0.0.0.0:$PORT --workers 2 --timeout 120
     ```
5. Add environment variables (see below)
6. Click "Create Web Service"

### Step 4: Configure Environment Variables

Add these in Render dashboard:

```bash
# Auto-generated by Render
DJANGO_SECRET_KEY=<auto-generated>
AGENT_INTERNAL_TOKEN=<auto-generated>

# Security Settings
DJANGO_DEBUG=0
DJANGO_ALLOWED_HOSTS=your-app.onrender.com
DJANGO_CORS_ALLOWED_ORIGINS=https://your-app.onrender.com
DJANGO_CSRF_TRUSTED_ORIGINS=https://your-app.onrender.com
DJANGO_SESSION_COOKIE_SECURE=1
DJANGO_CSRF_COOKIE_SECURE=1
DJANGO_SESSION_COOKIE_HTTPONLY=1
DJANGO_CSRF_COOKIE_HTTPONLY=1
DJANGO_USE_X_FORWARDED_HOST=1
DJANGO_TRUST_X_FORWARDED_PROTO=1

# Database
MONGODB_URI=mongodb+srv://user:pass@cluster.mongodb.net/
MONGODB_DB_NAME=agentdoc

# Storage
USE_LOCAL_STORAGE=1
MEDIA_ROOT=/opt/render/project/src/backend/media

# AI Configuration
GEMINI_API_KEY=your-gemini-api-key
GEMINI_MODEL=gemini/gemini-2.5-flash

# URLs
BACKEND_URL=https://your-app.onrender.com
REVIEW_PORTAL_URL=https://your-app.onrender.com/app/reviews

# Optional
RATE_LIMIT_PER_MINUTE=60
LOG_LEVEL=INFO
```

### Step 5: Verify Deployment

1. Wait for build to complete (5-10 minutes)
2. Check logs for errors
3. Visit your app URL
4. Test landing page
5. Test application login
6. Test document upload

## Troubleshooting

### Build Fails

**Issue**: Dependencies fail to install
**Solution**: Check Python version, update requirements.txt

**Issue**: Static files not collected
**Solution**: Verify STATIC_ROOT and STATICFILES_DIRS settings

### Runtime Errors

**Issue**: 500 Internal Server Error
**Solution**: Check logs, verify environment variables

**Issue**: Static files not loading
**Solution**: Run `python manage.py collectstatic` and check STATIC_URL

**Issue**: CORS errors
**Solution**: Verify DJANGO_CORS_ALLOWED_ORIGINS matches your domain

### Database Issues

**Issue**: Cannot connect to MongoDB
**Solution**: Check connection string, verify IP whitelist

**Issue**: Migration errors
**Solution**: Check MongoDB permissions, verify database name

### Performance Issues

**Issue**: Slow response times
**Solution**: 
- Increase worker count in gunicorn
- Optimize database queries
- Enable caching
- Consider upgrading Render plan

## Cost Optimization

### Free Tier Limits

- 750 hours/month (enough for 1 service)
- Spins down after 15 minutes of inactivity
- 512 MB RAM
- Shared CPU

### Tips

1. Use MongoDB Atlas free tier (M0)
2. Use local storage instead of S3
3. Optimize static files (compression)
4. Implement caching
5. Minimize background jobs

## Scaling

### When to Upgrade

- Consistent high traffic
- Need for always-on service
- Require more RAM/CPU
- Need multiple workers
- Require dedicated database

### Upgrade Path

1. Render Starter plan ($7/month)
2. Add Redis for caching
3. Upgrade MongoDB tier
4. Add CDN for static files
5. Implement load balancing
